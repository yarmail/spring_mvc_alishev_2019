Урок 32 Ограничения (Constraints) - NOT NULL, UNIQUE, PRIMARY KEY, CHECK

Пока в нашей таблице нет никаких ограничений, мы можем добавлять
любые значения (в том числе неправильные)

---

Самые популярные ограничения
NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, CHECK
PRIMARY KEY = UNIQUE + NOT NULL (Может быть только один первичный ключ)

---

Таким образом, у нас получаетмя двойная валидация - на морде и в БД
На фронте это делается, чтобы выдавать оповещения, в таблице это делается,
чтобы технически невозможно было поместить неправильное значение
