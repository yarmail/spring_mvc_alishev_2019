Урок 33 Отношения (связи) в БД. Внешний ключ (Foreign key)

Реляционная база данных
Relational Database
Relation - отношение, связь

Foreign key - внешний ключ - колонка,
которая позволяет установить связь между таблицами

---

Пример
Таблица Person
User_id | name | age
User_id - первичный ключ

Таблица Order
order_id | user_id | item_name
order_id - первичный ключ
user_id - внешний ключ

Заказы идут по порядку, но в нескольких заказах
может быть один и тот же покупатель.

Первичный ключ user_id может быть использован
в нескольких других таблицах как внешний ключ

---

Зачем разбивать данные на несколько таблиц
(нормализация таблиц)?
Чтобы избегать избыточности данных (Redundancy)
(денормализация таблиц)
