Урок 18: Протокол HTTP. Методы GET и POST.

Схема клиент - сервер

Клиент посылает HTTP запрос (request) на сервер
и получает HTTP ответ (response) сервера

---

Тело запроса (request)

Пример запроса
GET /wiki/java HTTP/1.1 - стартовая строка
Host: ru.wikipedia.org - Заголовки (Headers)
User-Agent: Mozilla/5.0
Accept: text/html
Пустая строка разделяет запросы и тело
k1=v1&k2=v2&k3=v3 - тело (Body) - может быть пустым

-

Метод: GET
Адрес: /wiki/java
Протокол: HTTP/1.1

---

В хроме мы все можем посмотреть в Инструментах
разработчика вкладке Network

---

Метод GET - идемпотентный (ничего не меняет на сервере)
Тело запроса пустое

---

GET - запрос
Параметры GET - запроса передаются в самом URL после
знака ? в формате ключ = значение
и разделяются знаком &

---

POST - запрос - от "публиковать"
Цель запроса - изменить что-то на сервере
(чаще всего добавить что-нибудь)
Часто используется для отправки данных с веб-форм)
Все параметры хранятся в теле запроса
Все параметры закодированы в теле запроса
Content-Type Может быть разный (JSON, XML, и т.д.)
Можем передавать разные данные
(пары ключ=значение, Java объекты, файлы и т.д.)

---

HTTP - ответ (response)
HTTP/1.1 200 OK - стартовая строка
Date: Wed, 17 Jun 2020 - headers (заголовки)
Server: Apache - headers (заголовки)
Content-Type: Text/html - headers (заголовки)

<h1>Hello world!</h1> - тело (Body)

---

Статусы ответа
200 - все ОК
3xx - редирект
4xx - ошибки клиента
5xx - ошибки сервера

---

HTTP - ответ (response)
Тип содержимого (Content-Type)
- text/html
- text/css
- text/xml
- application/json
